# Security Policy

## Supported Versions

The following versions of Urbindex receive security updates. Versions outside this range may contain unpatched vulnerabilities and should not be deployed in production environments.

| Version | Status      |
| ------- | ----------- |
| 5.1.x   | Supported   |
| 5.0.x   | Unsupported |
| 4.0.x   | Supported   |
| < 4.0   | Unsupported |

If you are running an unsupported version, upgrade to the latest stable release to ensure that you receive critical security fixes.

---

## Reporting a Vulnerability

We take security issues seriously and appreciate responsible disclosure.

If you believe you have found a vulnerability in Urbindex, please follow the process outlined below:

1. **Do not open a public GitHub issue.**
   Public issue trackers are not the appropriate place to disclose sensitive security findings.

2. **Send a detailed report via email:**
   **[security@numbpill3d.dev](mailto:voidrane@proton.me)**
   If possible, include:

   * A clear description of the vulnerability
   * Steps to reproduce the issue
   * Potential impact and severity
   * Any proof-of-concept code or screenshots

3. **Acknowledgment timeframe.**
   You can expect an acknowledgment within **72 hours**. If additional information is required, we will follow up promptly.

4. **Assessment and remediation.**
   Valid reports will be:

   * Reviewed and triaged
   * Assigned a severity rating
   * Patched in the next appropriate release
   * Documented in the projectâ€™s changelog when the fix is published

5. **Disclosure policy.**
   We request that you allow us reasonable time to investigate and deploy a fix before publicly disclosing the vulnerability.
